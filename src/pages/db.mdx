# {frontmatter.title}

## Fundamentals

- What is a relational database — tables, rows, columns, primary keys, foreign keys.
- Relationships — one-to-one, one-to-many, many-to-many.
- Normalization basics — avoid data redundancy (1NF, 2NF, 3NF).
- CRUD operations — Create, Read, Update, Delete.

## SQL Essentials

- SELECT, WHERE, ORDER BY, LIMIT — query data.
- INSERT, UPDATE, DELETE — manipulate data.
- JOIN — combine data from multiple tables (INNER, LEFT, RIGHT).
- GROUP BY, HAVING — aggregate and filter grouped data.
- Indexes — speed up queries on large tables.
- Views — save a query as a virtual table.
- Transactions — COMMIT, ROLLBACK, ensure data integrity.

## Tools

- MySQL — Most widely used open-source relational database.
- SQLite — Lightweight file-based database, no server needed, great for scripting.
- DBeaver — Universal database GUI client, works with almost any database.

## Python Integration

- sqlite3 — Built-in Python module to interact with SQLite databases.
- mysql-connector-python — Connect and query MySQL databases from Python.

## To Explore Later

- PostgreSQL — More advanced open-source database, widely used in production.
- SQLAlchemy — Python ORM to interact with databases without writing raw SQL.
- MongoDB — NoSQL document-based database, different paradigm from relational.
- Redis — In-memory key/value store, used for caching and fast data access.
- MariaDB — MySQL fork, often preferred in enterprise environments.
